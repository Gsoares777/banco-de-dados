Indexação

Indexação em bancos de dados é uma técnica para acelerar a busca e recuperação de dados. Ao criar um índice, o banco de dados constrói uma estrutura de dados adicional que facilita encontrar registros sem percorrer toda a tabela. Isso melhora a eficiência das consultas, especialmente em colunas frequentemente usadas. No entanto, índices podem aumentar o custo das operações de escrita, como inserções e atualizações, e usar mais espaço em disco. Índices comuns incluem árvores B e tabelas hash, e são criados em colunas-chave para otimizar o desempenho.

B-Tree

A B-Tree é uma estrutura de dados balanceada que armazena valores em uma árvore com vários níveis. Cada nó da árvore contém múltiplos valores e ponteiros para outros nós. Isso permite buscas rápidas e eficientes, pois a pesquisa é feita navegando da raiz até as folhas em um tempo logarítmico. A B-Tree é ideal para consultas que envolvem intervalos e ordenação. No entanto, a manutenção do índice pode ser mais lenta durante inserções e deleções, devido ao balanceamento da árvore.

Hash

O índice Hash utiliza uma função hash para mapear chaves para posições específicas em uma tabela. A função hash calcula um valor que determina onde os dados serão armazenados, permitindo acesso rápido e direto. Este tipo de índice é muito eficiente para buscas exatas, mas não é adequado para consultas que envolvem intervalos ou ordenação, e pode enfrentar problemas com colisões, onde diferentes chaves resultam no mesmo valor hash.

Bitmap

O índice Bitmap utiliza bits para representar a presença de valores em uma coluna. Cada valor distinto na coluna tem seu próprio bitmap, e cada bit representa uma linha na tabela, indicando se o valor está presente naquela linha. É muito eficiente para colunas com poucos valores distintos e consultas que combinam várias condições. Contudo, pode consumir muito espaço em tabelas com muitos valores distintos e não é ideal para dados com alta cardinalidade.
